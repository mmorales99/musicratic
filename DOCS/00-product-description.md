# Product Description

## Finished Product

This project is a gramola.

It is meant to be used in parties, coffee shops, bars, and car trips.

The MVP focuses on coffee shops and bars. In this context, we call them shops.

The finished product should feel like a clear, consistent, and maintainable music-sharing experience where each part has a defined role.

At the highest level, the system should:

- let a shop owner prepare the music session
- let visitors join the session from the shop
- let people suggest songs and vote on songs
- let the shop owner keep final authority over the session

## What This File Covers

This file stays at the highest level of abstraction.

It describes:

- what the product is
- why it exists
- where it is intended to be used
- what the MVP focuses on
- the general outcome the team is trying to achieve

It does not describe internal actors or detailed behavior. Those belong in the next files.
