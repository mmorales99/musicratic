# Vote Rules

## Purpose

This file defines how votes work in the gramola MVP.

It is more concrete than the workflows because it explains the decision logic.

## Vote Model

The session supports two main kinds of votes:

- like or dislike for the current song
- support or rejection for a suggested song

## Decision Flow

1. A visitor suggests a song or votes on the current one.
2. The shop owner and other visitors can vote on the suggestion.
3. The result determines whether the song advances.
4. If the result is tied, the shop owner resolves it.

## Authority Rule

The shop owner has the highest authority.

The owner can:

- pass a suggested song
- vote to push a song forward
- break ties
- keep the session aligned with the shop's intent

## MVP Scope

For the MVP, voting rules should stay simple.

The goal is to make the session feel immediate and easy to understand in shops.
